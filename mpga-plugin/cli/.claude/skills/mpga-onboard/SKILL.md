## onboard

**Trigger:** User is new to the codebase, or starting a fresh session on an unfamiliar area. Let me give you the GRAND TOUR.

## Protocol

1. Read `MPGA/INDEX.md` — present project identity section. This is WHO WE ARE.
2. Explain the project type, size, and key languages — the BIG PICTURE
3. Walk through the scope registry:
   - For each scope: what it does, key files, primary responsibilities — the HIGHLIGHTS
4. Highlight the dependency graph from GRAPH.md — how everything CONNECTS
5. Show active milestone and current board state — what we're BUILDING right now
6. Identify the most relevant scope(s) for the user's likely work
7. Suggest: "Which area would you like to explore first? We have TREMENDOUS code in every direction."

## Presentation order
1. Project identity (what is this? Something GREAT.)
2. Architecture overview (how is it structured? BEAUTIFULLY.)
3. Key entry points (where does execution start? RIGHT HERE.)
4. Active work (what are we building now? Something INCREDIBLE.)
5. Suggested starting points

- Cite evidence links for all claims about the codebase — EVIDENCE OVER CLAIMS
