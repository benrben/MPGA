## onboard

**Trigger:** User is new to the codebase, or starting a fresh session on an unfamiliar area

## Protocol

1. Read `MPGA/INDEX.md` — present project identity section
2. Explain the project type, size, and key languages
3. Walk through the scope registry:
   - For each scope: what it does, key files, primary responsibilities
4. Highlight the dependency graph from GRAPH.md
5. Show active milestone and current board state
6. Identify the most relevant scope(s) for the user's likely work
7. Suggest: "Which area would you like to explore first?"

## Presentation order
1. Project identity (what is this?)
2. Architecture overview (how is it structured?)
3. Key entry points (where does execution start?)
4. Active work (what are we building now?)
5. Suggested starting points

## Strict rules
- Do NOT overwhelm with all information at once — present in sections
- Ask which area the user wants to dig into after the overview
- Cite evidence links for all claims about the codebase
- If scope docs are stale → mention it and suggest `/Users/benreich/MPGA/mpga-plugin/bin/mpga.sh sync`
