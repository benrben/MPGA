# MPGA + Gemini CLI

Gemini CLI reads `AGENTS.md` from the project root for persistent context. MPGA generates this from your knowledge layer.

## Setup

```bash
# 1. Initialize MPGA (if not already done)
bash path/to/mpga-plugin/bin/mpga.sh init --from-existing
bash path/to/mpga-plugin/bin/mpga.sh sync

# 2. Export AGENTS.md
bash path/to/mpga-plugin/bin/mpga.sh export --gemini
```

This creates `AGENTS.md` in your project root.

## What gets exported

`AGENTS.md` contains:
- Agent instructions preamble (how to use MPGA's evidence format)
- Full `MPGA/INDEX.md` content — project identity, key files, conventions, scope registry

```markdown
# my-project — Agent Instructions (generated by MPGA)

This project uses MPGA for evidence-backed context engineering.

## Before starting any task
1. Read `MPGA/INDEX.md` for project overview and scope registry
2. Find relevant scope documents in `MPGA/scopes/`
3. Check evidence health: mpga evidence verify

## Evidence format
[E] filepath:startLine-endLine :: symbolName()   ← verified evidence
[Unknown] description                             ← explicit gap
[Stale:date] filepath                            ← needs re-verify

## Workflow
1. Explore: Read INDEX.md → find relevant scopes → read scope docs
2. Plan: Create evidence-backed plan citing file locations
3. Develop: TDD cycle — tests first, then implementation
4. Verify: Run drift check after each file change
5. Ship: Update scope evidence links, commit

---
# Project: my-project
## Identity
...
```

## Keep it updated

```bash
bash path/to/mpga-plugin/bin/mpga.sh sync && \
bash path/to/mpga-plugin/bin/mpga.sh export --gemini
```

## Usage in Gemini CLI

```bash
gemini "How does the auth middleware work?"
# Gemini reads AGENTS.md → understands to check MPGA/scopes/auth.md

gemini "Add a rate limiting endpoint to the API"
# Gemini reads AGENTS.md → understands evidence format → cites scope docs
```

## GEMINI.md alternative

Some Gemini CLI versions also read `GEMINI.md`. You can symlink:

```bash
ln -s AGENTS.md GEMINI.md
```

Or export directly:

```bash
bash path/to/mpga-plugin/bin/mpga.sh export --gemini
cp AGENTS.md GEMINI.md
```
